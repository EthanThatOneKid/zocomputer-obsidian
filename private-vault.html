Zo Computer: Obsidian: Private Vault

Your vault stays locked. Only you can see or edit your notes. Anyone who visits the app is redirected to a login page first.

First, tell Zo where your vault is. Replace the path with the absolute path to your notes folder. 

Stop the server if it is running. Then set VAULT_PATH to the absolute path of my Obsidian vault. The default is user-data/demo-vault if I do not set it. Do not restart yet.

Then tell Zo to protect the vault with a password. Provide a strong password when prompted. 

Now protect the vault with authentication.

Set AUTH_MODE to full.
Set OBSIDIAN_WEB_USERNAME to ethan.
Set OBSIDIAN_WEB_PASSWORD to a strong password. I will provide one.
Set OBSIDIAN_WEB_REALM to "Obsidian Web".

Then restart the server. After it starts, tell me the HTTP status code from http://127.0.0.1:3000. If it is 302, the login page is working.

The table below shows how each mode handles access. Full mode is the safest choice for a personal vault.

ModeReadWriteUse Case
fullAuth onlyAuth onlyPrivate vault
readonlyAnyoneAuth onlyPublic read
disabledAnyoneAnyoneFully open

If you do not set OBSIDIAN_WEB_PASSWORD, auth stays off no matter which mode you choose.
