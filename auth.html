Zo Computer: Obsidian: Lock It Down

The server supports three auth modes via the AUTH_MODE environment variable.

ModeReadWriteUse Case
fullAuth onlyAuth onlyPrivate vault
readonlyAnyoneAuth onlyPublic read
disabledAnyoneAnyoneFully open

Set these environment variables to configure auth: 

$ export AUTH_MODE=full
$ export OBSIDIAN_WEB_USERNAME=ethan
$ export OBSIDIAN_WEB_PASSWORD=your-secure-password
$ export OBSIDIAN_WEB_REALM="Obsidian Web"
$ npm start

With AUTH_MODE=full, visiting the app redirects to a sign-in form at /login. Authenticated users get an HttpOnly cookie session. 

$ curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:3000/
302

If OBSIDIAN_WEB_PASSWORD is unset, auth stays off regardless of AUTH_MODE.
