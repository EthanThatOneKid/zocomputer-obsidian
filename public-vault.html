Zo Computer: Obsidian: Public Vault

Anyone can read your notes without signing in. Only you can edit them after logging in.

Use with caution. Every note in your vault will be visible to anyone who visits the app. Do not use this mode if your vault contains private information like passwords, personal journals, or financial records.

First, tell Zo where your vault is. Replace the path with the absolute path to your notes folder. 

Stop the server if it is running. Then set VAULT_PATH to the absolute path of my Obsidian vault. The default is user-data/demo-vault if I do not set it. Do not restart yet.

Then tell Zo to allow public reading but require authentication for writing. Anyone can view, but only you can edit. 

Now configure the vault so anyone can view but only I can edit.

Set AUTH_MODE to readonly.
Set OBSIDIAN_WEB_USERNAME to ethan.
Set OBSIDIAN_WEB_PASSWORD to a strong password. I will provide one.
Set OBSIDIAN_WEB_REALM to "Obsidian Web".

Then restart the server. After it starts, verify that visiting http://127.0.0.1:3000 shows the app without needing to log in. Confirm that the edit controls are hidden for unauthenticated visitors.

The table below compares the three modes. Readonly mode hides edit controls for unauthenticated visitors and silently discards any write attempts.

readonlyAnyoneAuth onlyPublic read
